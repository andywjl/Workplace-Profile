# 楼宇档案系统优化建设 PRD

版本：V1.0
日期：2026-06-01
撰写角色：产品经理

## 1. 产品背景

当前楼宇档案系统已形成原型能力，覆盖中国区楼宇管理的三层分析视角：全国总览、区域分析、单楼宇档案。系统内置 100+ 楼宇、5 个管理区域、18 个管理维度、74 项指标，以及楼宇改进措施数据，能够通过达标率、趋势对比、区域热力图、单楼宇雷达图等方式辅助运营管理。

现阶段系统更偏“数据看板 + 指标填报 + 改进措施列表”，已经具备管理驾驶舱雏形，但在业务闭环、数据治理、权限分工、异常预警、改进追踪和系统扩展方面仍有优化空间。下一阶段建设目标应从“能看见问题”升级为“能定位问题、推动整改、验证成效、沉淀标准”。

## 2. 当前系统能力概览

### 2.1 已有核心模块

1. 首页与登录
   - 支持楼宇档案系统入口。
   - 支持演示登录与直接浏览。
   - 当前登录较轻量，未形成真实账号、角色和权限体系。

2. 全国总览
   - 支持区域、资产性质、周期筛选。
   - 展示职场总数、综合达标率、改进措施数、数据填报率、未达标楼宇。
   - 展示四大核心维度：环境安全、温度适宜、照明亮堂、空气清新。
   - 支持维度趋势对比、维度区域热力图、18 维度达标率、自持园区与租赁职场对比。
   - 支持区域概览、改进措施汇总、楼宇列表。

3. 区域分析
   - 支持按京区、北区、东区、西区、南区下钻。
   - 展示区域 KPI、区域维度达标率、资产性质对比、区域楼宇排行、区域改进措施。

4. 单楼宇档案
   - 展示楼宇基础信息、画像分析、全维度雷达图。
   - 支持按周期查看和编辑指标值。
   - 展示维度详情、指标目标、实际值、达标率。
   - 展示楼宇改进方案。
   - 展示资产管理、能源管理、费用分析等辅助模块。

5. 指标与改进措施
   - 已配置 18 个维度、74 个指标。
   - 支持指标实际值录入与保存。
   - 支持创建、更新改进措施。
   - 改进措施包含名称、状态、维度、描述、阶段、预算、预期效果、效果验证等字段。

### 2.2 现有问题判断

1. 业务闭环不完整
   - 当前系统能展示达标率和改进措施，但“指标异常 -> 自动生成问题 -> 责任人认领 -> 整改推进 -> 效果验证 -> 复盘沉淀”的闭环还不完整。

2. 指标填报体验不足
   - 指标值以手工录入为主，缺少批量导入、模板校验、异常值提示、填报任务提醒、审批确认等能力。

3. 权限和角色体系较弱
   - 当前登录逻辑偏演示，缺少总部管理员、区域负责人、楼宇负责人、专项 POC、只读访客等角色分工。

4. 指标口径和数据质量需要加强
   - 指标定义、目标值、适用条件、数据来源已存在，但缺少口径版本管理、填报责任人、数据可信度、缺失原因和审计记录。

5. 改进措施管理较浅
   - 已有措施列表与状态更新，但缺少优先级、负责人、里程碑、风险、附件、投入产出、逾期预警和验收标准。

6. 决策洞察仍偏展示
   - 当前图表可视化较丰富，但缺少自动诊断、根因归类、跨楼宇共性问题识别、整改建议排序和管理摘要。

7. 全球化扩展仍为空
   - Non-China 当前为占位页面，未来若扩展海外区域，需要考虑指标本地化、单位换算、时区、语言、合规差异。

## 3. 产品定位

楼宇档案系统是面向职场与楼宇运营管理团队的数字化管理平台，用于统一楼宇基础档案、指标评价、运营风险识别、改进措施跟踪和管理决策分析，帮助总部、区域和楼宇团队以同一套标准推进服务质量提升。

产品关键词：
- 统一标准：沉淀楼宇管理维度、指标口径、目标值和评价规则。
- 可视管理：从全国、区域、楼宇三层视角快速识别问题。
- 闭环整改：将未达标指标转化为可跟踪的措施、责任和验收。
- 数据可信：提升填报完整性、准确性、可追溯性。
- 经营联动：将服务质量、能源、费用、资产和体验关联分析。

## 5. 产品目标

### 5.1 业务目标

1. 建立全国楼宇统一评价标准，覆盖 18 个维度和核心运营指标。
2. 将楼宇质量管理从静态看板升级为问题闭环管理。
3. 提升数据填报完整率和及时率，降低人工统计成本。
4. 帮助总部和区域识别重点楼宇、重点维度、共性短板。
5. 支持周期性复盘，量化改进措施对达标率、体验和成本的影响。

### 5.2 产品指标

| 指标 | 目标值 |
| --- | --- |
| 周期数据填报完整率 | ≥ 95% |
| 指标异常自动识别覆盖率 | ≥ 90% |
| 未达标指标生成整改任务覆盖率 | ≥ 90% |
| 整改措施按期完成率 | ≥ 85% |
| 改进措施效果验证完成率 | ≥ 80% |
| 区域负责人周活跃率 | ≥ 80% |
| 单楼宇档案完整率 | ≥ 95% |

## 7. 核心用户旅程

### 7.1 总部管理者

1. 进入全国总览，查看本周期综合达标率、数据填报率、未达标楼宇数。
2. 查看四大核心维度与 18 维度趋势，识别下降明显或低于目标的维度。
3. 点击热力图下钻到区域，再进入具体楼宇。
4. 查看单楼宇未达标指标和关联改进措施。
5. 对重点问题发起督办，要求区域或楼宇负责人限期整改。
6. 周期末导出复盘报告，查看措施完成率和效果验证。

### 7.2 区域负责人

1. 进入区域分析页，查看区域内楼宇排行。
2. 筛选未达标楼宇和高风险维度。
3. 对系统自动识别的问题进行确认、合并或忽略。
4. 指派楼宇负责人和专项 POC，设定整改期限。
5. 跟踪逾期、待验收、效果未达预期的措施。
6. 在区域周会中导出区域整改清单。

### 7.3 楼宇负责人

1. 收到周期填报任务。
2. 进入单楼宇档案，逐项填报指标或导入模板。
3. 系统提示缺失值、异常值、口径说明和历史对比。
4. 对未达标指标创建或关联改进措施。
5. 上传整改过程说明、预算、预计完成时间和验收材料。
6. 整改完成后提交专项 POC 验收。

## 8. 功能需求

### 8.1 账号、角色与权限

#### 8.1.1 角色管理

系统应支持以下角色：
- 超级管理员：配置系统、指标、角色、区域、楼宇。
- 总部管理员：查看全国数据，发起督办，导出报告。
- 区域负责人：管理所属区域楼宇、指标和措施。
- 楼宇负责人：维护指定楼宇数据和措施。
- 专项 POC：维护专项指标口径，审核专项整改效果。
- 只读用户：查看授权范围内数据。

#### 8.1.2 权限范围

| 操作 | 总部管理员 | 区域负责人 | 楼宇负责人 | 专项 POC | 只读用户 |
| --- | --- | --- | --- | --- | --- |
| 查看全国总览 | 是 | 可配置 | 否 | 可配置 | 可配置 |
| 查看区域数据 | 是 | 本区域 | 所属楼宇所在区域 | 相关维度 | 可配置 |
| 编辑楼宇档案 | 是 | 本区域 | 所属楼宇 | 否 | 否 |
| 编辑指标值 | 是 | 本区域 | 所属楼宇 | 相关维度可审核 | 否 |
| 创建整改措施 | 是 | 本区域 | 所属楼宇 | 可建议 | 否 |
| 验收整改措施 | 是 | 本区域 | 否 | 相关维度 | 否 |
| 配置指标口径 | 是 | 否 | 否 | 可建议 | 否 |

#### 8.1.3 登录与安全

需求：
- 接入真实账号体系或企业 SSO。
- 登录态有效期可配置。
- 记录关键操作日志。
- 支持按角色和数据范围控制接口访问。

验收标准：
- 未登录用户不能访问业务接口。
- 楼宇负责人无法编辑非授权楼宇。
- 所有指标值和措施状态变更均可追溯到操作人和时间。

### 8.2 楼宇档案管理

#### 8.2.1 基础档案

现有字段基础上建议补充：
- 楼宇编码
- 楼宇状态：筹备中、运营中、退租中、已关闭
- 入驻业务类型
- 开放日期、租约起止日期
- 物业方、服务商、楼宇负责人
- 可用工位数、实际工位数、入驻人数
- 建筑面积、使用面积、楼层范围
- 是否重点楼宇、是否核心城市楼宇

#### 8.2.2 档案完整度

系统应自动计算楼宇档案完整度：
- 必填字段完整率。
- 关键附件完整率。
- 指标填报完整率。
- 责任人配置完整率。

验收标准：
- 楼宇列表支持按档案完整度排序。
- 单楼宇页展示缺失字段清单。
- 档案完整度低于阈值时，在区域页提示。

### 8.3 指标体系管理

#### 8.3.1 指标配置

支持后台维护：
- 维度名称、排序、负责人。
- 指标名称、定义、目标值、目标类型、单位。
- 数据来源、填报周期、适用范围、行业标杆。
- 指标权重、是否核心指标、是否参与总分。
- 口径说明、示例、常见错误。
- 指标版本号和生效周期。

#### 8.3.2 指标权重

当前达标率计算接近平均法，建议引入权重：
- 全国综合分 = 各维度得分按维度权重汇总。
- 维度得分 = 指标得分按指标权重汇总。
- 四大核心维度可设置更高权重。
- 支持按总部统一权重和区域差异权重两种模式。

默认建议：
- 四大核心维度：环境安全、温度适宜、照明亮堂、空气清新，共 40%。
- 其他 14 个维度，共 60%。
- 安全类指标应具备一票否决能力。

#### 8.3.3 适用性规则

部分指标并非所有楼宇适用，例如电梯、停车、直饮水、仓储等。系统应支持：
- 按楼宇类型自动判断是否适用。
- 手工标记不适用并填写原因。
- 不适用指标不计入达标率分母。
- 不适用原因可被总部审核。

验收标准：
- 指标不适用时，楼宇得分不会被错误拉低。
- 不适用记录包含原因、提交人、审核人和时间。

### 8.4 数据填报与质量校验

#### 8.4.1 填报任务

系统应按周期自动生成填报任务：
- 月度指标
- 季度指标
- 半年度指标
- 年度指标

任务字段：
- 填报周期
- 负责楼宇
- 待填指标数量
- 截止时间
- 当前状态：未开始、填报中、待审核、已完成、退回
- 填报人、审核人

#### 8.4.2 批量导入

支持 Excel/CSV 模板导入：
- 下载标准模板。
- 导入前预校验。
- 展示错误行、错误字段、错误原因。
- 支持部分成功导入。
- 导入后生成操作记录。

#### 8.4.3 数据质量校验

系统应支持：
- 缺失值校验。
- 数值范围校验。
- 同比/环比异常波动校验。
- 单位校验。
- 指标口径提示。
- 同类楼宇横向对比异常提示。

异常等级：
- 阻断：明显非法值，不允许提交。
- 警告：疑似异常，可填写说明后提交。
- 提醒：建议关注，不影响提交。

验收标准：
- 填报完成前展示缺失指标数量。
- 异常值提交必须填写说明。
- 总部可查看各区域数据质量排行。

### 8.5 达标评价与异常识别

#### 8.5.1 评价逻辑

系统应支持以下目标类型：
- 上限型：实际值小于等于目标值为达标。
- 下限型：实际值大于等于目标值为达标。
- 固定型：实际值等于目标值为达标。
- 趋势型：相对上一周期改善为达标。
- 区间型：实际值落在目标区间内为达标。
- 复合型：同时满足多个条件才达标。

建议补充区间型和复合型，以解决温湿度、男女厕位配比、人均新风量等场景。

#### 8.5.2 风险分级

对未达标指标进行分级：
- P0：安全或业务连续性重大风险，需立即处理。
- P1：核心体验或关键设备风险，需限期处理。
- P2：运营效率或体验优化问题，纳入计划处理。
- P3：低频或观察类问题，持续关注。

分级依据：
- 指标类别。
- 偏离目标程度。
- 影响人数。
- 是否连续多周期未达标。
- 是否核心城市或重点楼宇。

#### 8.5.3 自动问题生成

当指标未达标时，系统自动生成问题项：
- 问题标题：楼宇 + 维度 + 指标异常。
- 问题描述：实际值、目标值、偏离程度、历史趋势。
- 风险等级。
- 建议责任角色。
- 建议整改期限。
- 可关联已有改进措施。

验收标准：
- 未达标指标可一键转为问题。
- 同一楼宇同一指标连续异常时自动合并为同一问题。
- 问题关闭必须关联整改措施或填写豁免原因。

### 8.6 改进措施闭环

#### 8.6.1 措施字段优化

在当前措施字段基础上补充：
- 措施编号
- 问题来源：系统自动生成、人工创建、总部督办
- 关联问题 ID
- 负责人
- 协同人
- 优先级
- 当前进度百分比
- 里程碑
- 预计收益类型：体验、安全、成本、效率、合规
- 实际收益
- 验收人
- 验收状态：待验收、通过、退回
- 附件与图片
- 逾期原因

#### 8.6.2 措施状态流转

建议状态：
1. 待确认
2. 未开始
3. 进行中
4. 待验收
5. 已完成
6. 已关闭
7. 已延期
8. 已取消

流转规则：
- 自动生成的问题默认进入待确认。
- 负责人确认后进入未开始或进行中。
- 提交完成后进入待验收。
- 验收通过进入已完成。
- 达到效果验证周期后进入已关闭。
- 验收不通过退回进行中。

#### 8.6.3 效果验证

系统应支持整改前后对比：
- 关联指标整改前实际值。
- 整改后实际值。
- 达标率变化。
- 投入预算。
- 实际花费。
- 影响人数。
- 验证结论。

验收标准：
- 已完成措施必须填写效果验证。
- 没有效果验证的措施不计入“闭环完成”。
- 单楼宇页可查看措施对维度得分的影响。

### 8.7 管理驾驶舱优化

#### 8.7.1 全国总览增强

新增模块：
- 本周期重点风险 TOP 10。
- 连续两期未改善楼宇。
- 四大核心维度红黄绿灯。
- 整改逾期措施数。

## 11. 关键规则

### 11.1 达标率计算

建议统一输出三类分数：
- 指标得分：单个指标的完成情况。
- 维度得分：维度内指标加权结果。
- 楼宇健康度：维度得分、风险等级、数据质量、整改闭环综合结果。

### 11.2 数据锁定

规则：
- 周期填报审核通过后锁定。
- 锁定后修改需发起更正申请。
- 更正需记录原因和审批人。

### 11.3 一票否决

如发生一级及以上环境安全事件、重大业务中断等，楼宇健康度应触发风险标记，即使其他指标平均分较高也不能显示为健康。

### 11.4 整改逾期

规则：
- 到期未进入待验收或已完成状态即为逾期。
- 逾期 7 天以上自动升级提醒区域负责人。
- 逾期 14 天以上进入总部督办池。

## 12. 非功能需求

### 12.1 性能

- 全国总览首屏加载时间 ≤ 3 秒。
- 区域分析加载时间 ≤ 3 秒。
- 单楼宇档案加载时间 ≤ 2 秒。
- 1000 栋楼宇、100 个指标、8 个周期数据量下保持可用。

### 12.2 可用性

- 表格支持搜索、排序、筛选、导出。
- 图表点击后必须有明确下钻路径。
- 保存、提交、导入等操作需有成功或失败反馈。
- 重要操作需二次确认。

### 12.3 安全

- 接口按角色鉴权。
- 敏感操作记录审计日志。
- 数据导出受权限控制。
- 防止越权访问楼宇和区域数据。

### 12.4 可扩展性

- 指标体系支持版本化。
- 支持新增区域、海外区域、资产类型。
- 支持未来对接工单系统、能耗系统、设备系统和满意度系统。

## 15. 验收标准汇总

1. 用户登录后仅能查看和操作授权范围内的数据。
2. 每个周期可自动生成填报任务，并能查看进度。
3. 指标填报支持校验，异常值需说明后提交。
4. 未达标指标可自动生成问题，并关联整改措施。
5. 改进措施支持负责人、期限、状态流转、验收和效果验证。
6. 全国、区域、楼宇三个层级均可查看风险、趋势和整改进度。
7. 关键数据变更均记录操作日志。
8. 周期数据审核后可锁定，修改需走更正流程。
9. 可导出全国、区域、楼宇和整改清单报告。
10. 系统在现有数据规模扩大 10 倍后仍满足核心页面性能要求。

## 16. 待确认问题

1. 是否已有企业账号或 SSO 接入标准。
2. 各区域和楼宇真实责任人名单是否已维护。

4. 哪些指标允许自动从外部系统同步，哪些必须人工填报。

7. 海外区域是否在 2026 年内进入建设范围。

## 17. 总结

楼宇档案系统当前已经具备清晰的三层看板结构和较完整的指标雏形。下一阶段优化的关键不在于继续增加图表，而在于补齐“数据可信、问题识别、责任分派、整改验收、效果复盘”的运营闭环。建议以 V1.1 为优先版本，先建立真实角色权限、填报任务、异常问题和措施流转，再逐步推进数据治理、管理洞察和系统集成。
